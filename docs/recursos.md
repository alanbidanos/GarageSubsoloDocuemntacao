## 3. Principais Recursos e Funcionalidades

O sistema da Garage Subsolo oferece um conjunto de funcionalidades voltadas à gestão de serviços, vendas e operações da oficina. Os principais recursos incluem:

- **Aluguel de bicicletas:**  
  Os clientes podem verificar modelos disponíveis, consultar horários e realizar reservas diretamente pelo site.

- **Venda de bicicletas:**  
  Marketplace integrado que permite a comercialização de bicicletas novas e usadas.

- **Venda de peças e acessórios de ciclismo:**  
  Catálogo completo de peças como pedais, freios, amortecedores, entre outros, integrando estoque físico e online.

- **Atendimento emergencial 24h:**  
  Sistema de solicitações de emergência para reparos imediatos, disponível a qualquer hora do dia.


## 3.1 Requisitos Funcionais

Os requisitos funcionais definem **o que o sistema deve fazer**:

1. **RF001 – Requisição de reparo:**  
   O usuário deve poder solicitar um reparo, escolher o tipo de serviço e agendar a execução.

2. **RF002 – Chamada de emergência 24h:**  
   O sistema deve permitir que o usuário abra solicitações emergenciais a qualquer horário.

3. **RF003 – E-commerce e marketplace:**  
   O sistema deve permitir compras de bicicletas, peças e acessórios em um ambiente virtual integrado.

4. **RF004 – Consulta de disponibilidade de bicicletas:**  
   O sistema deve exibir modelos disponíveis e datas livres para aluguel.

5. **RF005 – Aluguel de bicicletas:**  
   O sistema deve permitir reservar, cadastrar e acompanhar o aluguel de bicicletas.

6. **RF006 – Venda de bicicletas:**  
   O sistema deve permitir a compra de bicicletas novas ou usadas.

7. **RF007 – Venda de peças e acessórios:**  
   O sistema deve permitir a compra de peças e acessórios integrados ao estoque.


## 3.2 Requisitos Não Funcionais

Os requisitos não funcionais definem **como o sistema deve se comportar**:

1. **RNF001 – Hierarquia de permissões:**  
   O sistema deve organizar suas ferramentas com diferentes níveis de acesso (cliente, mecânico e administrador).

2. **RNF002 – Segurança de senhas:**  
   As senhas devem ser armazenadas de forma criptografada.

3. **RNF003 – Segurança física dos dados:**  
   O servidor físico deve possuir acesso restrito e protegido.

4. **RNF004 – Responsividade:**  
   O sistema deve funcionar corretamente em celulares, tablets e computadores.
